# Contributing to Core Spring Framework

First of all, thank you for taking the time to contribute! 🎉

We welcome contributions that improve the quality of this repository, whether it's fixing bugs, improving documentation, or adding new Spring Framework examples.

---

## How Can You Contribute?

You can contribute in several ways:

- Add new Spring Core examples
- Improve existing code
- Fix bugs
- Improve documentation
- Refactor code for better readability
- Add comments explaining concepts
- Suggest improvements

---

## Getting Started

### 1. Fork the Repository

Click the **Fork** button on GitHub.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/core-spring-framework.git
```

### 3. Navigate to the Project

```bash
cd core-spring-framework
```

### 4. Create a New Branch

```bash
git checkout -b feature/your-feature-name
```

Example:

```bash
git checkout -b feature/bean-lifecycle-example
```

---

## Coding Guidelines

Please follow these guidelines:

- Write clean and readable Java code.
- Follow Java naming conventions.
- Keep methods short and meaningful.
- Add comments where necessary.
- Avoid duplicate code.
- Keep commits focused on a single change.

---

## Commit Messages

Use meaningful commit messages.

Good examples:

```text
Add constructor injection example

Fix bean lifecycle documentation

Improve XML configuration example

Refactor package structure
```

Avoid messages like:

```text
Update

Changes

Fix

Done
```

---

## Testing

Before submitting your changes:

- Ensure the project builds successfully.
- Verify that existing examples continue to work.
- Test any new examples.

Build using Maven:

```bash
mvn clean install
```

---

## Pull Request Process

1. Fork the repository.
2. Create a feature branch.
3. Make your changes.
4. Commit your work.
5. Push to your fork.

```bash
git push origin feature/your-feature-name
```

6. Open a Pull Request.
7. Describe what changes you made and why.

---

## Reporting Issues

If you find a bug or have a suggestion:

- Check if an issue already exists.
- If not, create a new issue.
- Include steps to reproduce the problem.
- Provide screenshots or logs if applicable.

---

## Code Style

Please maintain:

- Proper indentation
- Meaningful variable names
- Organized package structure
- Java coding standards
- Clean formatting

---

## Questions?

If you have any questions, feel free to open an issue.

---

Thank you for helping improve this project!
